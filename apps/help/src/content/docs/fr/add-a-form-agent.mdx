Un **agent de formulaire** aide un utilisateur à **remplir un formulaire par la
conversation**. Au lieu d'afficher des champs statiques, il pose les questions une à
une et collecte des **réponses structurées** conformes à un schéma que vous
définissez — combinant l'expérience d'un chat avec la sortie structurée d'un
[agent d'extraction](/fr/add-an-extraction-agent).

Les agents appartiennent à votre **espace de travail** et se créent depuis la carte
**Nouvel agent** de la vue d'ensemble (ou le **+** à côté d'**Agents** dans la barre
latérale gauche). C'est le choix du type **Formulaire** qui transforme l'agent en
formulaire guidé.

## Comment fonctionne un agent de formulaire

Vous définissez les **champs à collecter** dans l'onglet **Formulaire** (par exemple
`full_name` et `email`), un **Message d'accueil** par lequel l'agent commence, et des
**Instructions** décrivant comment il doit guider l'utilisateur. Un agent de
formulaire dispose de trois onglets de configuration — **Général**, **Modèle** et
**Formulaire** — et **chaque onglet est un formulaire à part qui s'enregistre
séparément** (l'éditeur vous avertit avant de quitter un onglet non enregistré).

## Étape par étape

### 1. Ouvrir la boîte de dialogue Nouvel agent

Sur la vue d'ensemble de l'espace de travail, repérez la carte **Nouvel agent** et
cliquez sur **Créer** (vous pouvez aussi utiliser le **+** à côté d'**Agents** dans
la barre latérale gauche).

### 2. Choisir le type Formulaire

Dans la boîte de dialogue **Nouvel agent**, sous **Type d'agent**, sélectionnez
**Formulaire**. Le type détermine ce que fait l'agent : choisissez-le avant de
continuer.

### 3. Nommer l'agent et le créer

Saisissez un **Nom** (au moins 3 caractères) dans le champ **Nom**, puis cliquez sur
**Créer**. Votre agent est créé avec des valeurs par défaut pertinentes et s'ouvre
dans l'éditeur.

### 4. Régler l'essentiel dans Général

L'éditeur s'ouvre sur l'onglet **Général**. Définissez :

- le **Nom** — la façon dont l'agent apparaît dans votre espace de travail ;
- la **Langue** — la langue dans laquelle l'agent travaille ;
- le **Message d'accueil** — le premier message envoyé au début d'une conversation ;
- les **Instructions** — comment l'agent doit guider l'utilisateur à travers le
  formulaire.

Cliquez sur **Enregistrer**.

### 5. Choisir le modèle dans Modèle

Ouvrez l'onglet **Modèle**, choisissez le **Modèle** qui pilote l'agent et une
**Température** (une valeur basse donne des questions plus stables et prévisibles).
**Enregistrez**.

### 6. Construire le formulaire dans Formulaire

Ouvrez l'onglet **Formulaire** et construisez les champs que l'agent collecte
**visuellement** — sans JSON. Cliquez sur **Ajouter un champ** et, pour chaque champ,
définissez :

- **Nom du champ** — la clé sous laquelle la réponse est stockée (par ex.
  `full_name`).
- **Type** — **Texte**, **Nombre** ou **Oui / Non**.
- **Description** — la question ou l'indice que l'agent utilise pour demander ce
  champ.
- **Obligatoire** — activez-le pour les réponses obligatoires.

Supprimez un champ avec son icône **corbeille**.

Pour contrôler l'ordre dans lequel l'agent pose les questions, activez **Poser les
questions dans cet ordre** et **glissez** les lignes dans l'ordre souhaité ;
désactivez-le pour laisser l'assistant choisir l'ordre lui-même. Besoin d'un contrôle
plus fin ? Cliquez sur **Mode avancé** (en haut à droite) pour éditer le JSON brut, et
**Éditeur visuel** pour revenir. Cliquez sur **Enregistrer** — et rappelez-vous que
chaque onglet s'enregistre séparément.

## Historique des versions

Chaque enregistrement crée une nouvelle **version** de l'agent. Ouvrez **Historique**
(en haut à droite de l'éditeur, avec le **v numéro** actuel) pour parcourir les
versions passées dans un panneau latéral, **comparer** les changements
(**Changements de cette version** ou **Comparer avec l'actuelle**), et **Restaurer
cette version** — ce qui copie ces paramètres comme nouvelle version actuelle sans
rien supprimer. Voir [Historique des versions](/fr/agent-version-history) pour le
guide complet.

## Conseils

- Gardez des **descriptions** de champs claires — l'agent s'en sert pour formuler
  chaque question et pour décider quoi enregistrer.
- Utilisez **Poser les questions dans cet ordre** quand la séquence compte (par ex.
  demander un nom avant une préférence) ; laissez-le désactivé pour que l'agent
  s'adapte.
- Le **Message d'accueil** donne le ton ; rédigez-le dans la même langue que la
  **Langue** de l'agent.
- Utilisez une **Température basse** (par exemple `0.2`) pour que l'agent pose des
  questions cohérentes plutôt que d'improviser.
- Chaque onglet s'enregistre séparément — les modifications d'un onglet ne sont pas
  conservées si vous passez à un autre sans enregistrer.

## Dépannage

- **Le bouton Créer est désactivé** — le **Nom** doit comporter au moins 3
  caractères.
- **Je ne vois pas le type de champ dont j'ai besoin** — l'éditeur visuel couvre
  **Texte**, **Nombre** et **Oui / Non** ; utilisez le **Mode avancé** (JSON brut)
  pour des listes ou des objets imbriqués.
- **L'agent pose les questions dans le mauvais ordre** — activez **Poser les questions
  dans cet ordre** et glissez les champs dans la séquence voulue.
- **Mes modifications ont disparu** — chaque onglet s'enregistre séparément ; cliquez
  sur **Enregistrer** avant de changer d'onglet (l'éditeur vous avertit).
- **Je ne vois pas d'onglet Formulaire** — assurez-vous d'avoir créé un agent
  **Formulaire** (les agents conversationnels discutent, les agents d'extraction
  affichent plutôt un onglet **Sortie**).
