Une **source web** est un site web que vous explorez (crawl) pour indexer son
contenu et l'ajouter aux connaissances de vos agents. Une fois une source
explorée et prête, les agents de l'espace de travail peuvent s'appuyer sur ses
pages pour répondre aux utilisateurs.

Les sites web se trouvent dans **Sources**, aux côtés des [Documents](/fr/documents) et des
[Bibliothèques de ressources](/fr/resource-libraries), et appartiennent à l'**espace de travail**.

> **Fonctionnalité optionnelle — à faire activer.** Les sites web s'activent espace
> de travail par espace de travail. Si vous ne voyez pas **Sites web** sous
> **Sources** dans Studio, c'est que la fonctionnalité n'est pas encore activée pour
> votre espace de travail — contactez l'équipe Bayes pour en faire la demande.

## Comment les agents utilisent les sources web

Une source web explorée est indexée comme connaissance dans l'espace de travail.
Une fois son **statut des embeddings** à **Prêt**, les agents peuvent en récupérer
les pages pertinentes pour répondre aux utilisateurs. Utilisez les **étiquettes**
pour organiser vos sources et contrôler leur usage. À noter : contrairement à un
[document](/fr/documents) téléversé, une **source web explorée n'est jamais proposée
au téléchargement** aux utilisateurs — ses pages n'ont pas de fichier stocké — donc
l'étiquette réservée `public-documents` n'a aucun effet de téléchargement sur une
source web.

## Étape par étape

### 1. Ouvrir Sites web

Dans **Studio**, ouvrez la barre latérale de gauche, dépliez la section
**Sources**, puis sélectionnez **Sites web**.

### 2. Explorer un site web

1. Cliquez sur **Explorer un site web** (en haut à droite de la page).
2. Dans la boîte de dialogue **Explorer un site web**, saisissez l'**URL du site**
   (par exemple `https://exemple.com`). L'intégralité du site est indexée en
   arrière-plan.
3. Donnez éventuellement un **Nom** à la source (sinon le site est utilisé).
4. Cliquez sur **Lancer l'exploration**.

La nouvelle source apparaît aussitôt dans le tableau.

### 3. Attendre qu'elle soit prête

Chaque source affiche un **Statut des embeddings** qui évolue ainsi :

- **Exploration** — les pages sont en cours de récupération. Dès que le décompte
  commence, il affiche **Exploration — N pages**.
- **Indexation** — les pages récupérées sont en cours d'indexation.
- **Prêt** — l'indexation est terminée ; la source est pleinement exploitable par
  vos agents.
- **Échoué** — l'indexation a échoué ; relancez-la avec **Relancer le traitement**
  (voir l'étape 4).

Seule une source **Prêt** est pleinement exploitable par vos agents. Le tableau
indique aussi le nombre de **Pages** explorées et la date de dernière mise à jour.

### 4. Gérer une source

Ouvrez le menu **⋮** en fin de ligne d'une source pour :

- **Voir** — consulter les détails de la source.
- **Modifier** — renommer la source et ajouter des étiquettes (voir ci-dessous).
- **Relancer le traitement** — relancer l'indexation d'une source. Cet élément
  n'apparaît **que** lorsque le **Statut des embeddings** est **Échoué**, et
  démarre aussitôt (sans confirmation).
- **Arrêter l'exploration** — annuler une exploration encore en cours. Cet élément
  n'apparaît **que tant que la source est en Exploration** et prend effet aussitôt
  (sans confirmation).
- **Ré-explorer le site** — relancer l'exploration pour récupérer les changements.
  Affiché lorsque la source n'est **pas** en cours d'exploration. (**Arrêter
  l'exploration** et **Ré-explorer le site** n'apparaissent jamais ensemble — l'un
  remplace l'autre selon qu'une exploration est en cours.)
- **Supprimer** — retirer la source et son contenu indexé.

### 5. Voir les pages explorées

Cliquez sur la **flèche** (chevron) au début de la ligne d'une source pour la
déplier et lister toutes les pages explorées, chacune renvoyant vers son URL.

### 6. Étiqueter une source

Les étiquettes permettent de regrouper et de contrôler les sources.

1. Dans le menu **⋮** de la source, choisissez **Modifier**.
2. Dans la boîte de dialogue, cliquez sur **Ajouter une étiquette** et choisissez-en une.
3. Cliquez sur **Mettre à jour**.

L'étiquette apparaît alors dans la colonne **Tags** de la source.

### 7. Créer et gérer les étiquettes

1. Sur la page **Sites web**, cliquez sur **Étiquettes** (en haut à droite) pour
   ouvrir le panneau des étiquettes.
2. Cliquez sur **Créer une étiquette**.
3. Saisissez un **Nom** et une **Description** facultative, puis cliquez sur **Créer**.
4. Depuis ce panneau, vous pouvez **Modifier** ou **Supprimer** une étiquette à
   tout moment.

> L'étiquette réservée **`public-documents`** ne rend téléchargeables que les
> **[Documents](/fr/documents) téléversés**, dans une conversation. Une source web
> explorée ne peut pas être téléchargée même si elle porte l'étiquette, car ses
> pages n'ont pas de fichier stocké.

## Conseils

- Explorez les sections les plus pertinentes d'un site — tout le site est indexé,
  donc un site ciblé et bien structuré donne les meilleures réponses.
- **Ré-explorez** après une modification du site pour que vos agents restent à jour.
- Utilisez les étiquettes pour organiser de grands ensembles de sources et
  contrôler la façon dont vos agents les utilisent.

## Dépannage

- **Un agent n'utilise pas une source** — vérifiez que son **Statut des embeddings**
  est **Prêt** ; une source encore en **Exploration** ou en **Indexation** n'est pas
  encore disponible.
- **Le site a changé mais les réponses sont périmées** — ouvrez le menu **⋮** et
  choisissez **Ré-explorer le site**.
- **Une exploration est bloquée ou lancée par erreur** — ouvrez le menu **⋮** et
  choisissez **Arrêter l'exploration** (affiché uniquement tant que la source est
  encore en **Exploration**).
- **Une source affiche Échoué** — ouvrez le menu **⋮** et choisissez **Relancer le
  traitement** pour relancer l'indexation.
- **L'exploration ne démarre pas** — l'**URL du site** doit être une adresse
  `https://` valide.
