Un **agent d'extraction** lit un document que vous lui fournissez et renvoie des
**données structurées** — un objet JSON dont vous définissez les champs à l'avance.
Au lieu de discuter, il remplit un schéma : vous décrivez les champs souhaités, et
l'agent en extrait les valeurs de chaque document.

Les agents appartiennent à votre **espace de travail** et se créent depuis la carte
**Nouvel agent** de la vue d'ensemble de l'espace de travail (ou le **+** à côté
d'**Agents** dans la barre latérale gauche). C'est le choix du type **Extraction**
qui fait que l'agent extrait des données plutôt que de tenir une conversation.

## Comment fonctionne un agent d'extraction

Vous définissez les **champs à capturer** dans l'onglet **Sortie** (par exemple
`title` et `summary`) et rédigez des **Instructions** indiquant à l'agent quoi
extraire. Lorsque l'agent s'exécute sur un document, il renvoie un objet JSON
conforme à ces champs. Un agent d'extraction dispose de trois onglets de
configuration — **Général**, **Modèle** et **Sortie** — aucun réglage de conversation
ou de message d'accueil — et **chaque onglet s'enregistre séparément** (l'éditeur
vous avertit avant de quitter un onglet non enregistré).

## Étape par étape

### 1. Ouvrir la boîte de dialogue Nouvel agent

Sur la vue d'ensemble de l'espace de travail, repérez la carte **Nouvel agent** et
cliquez sur **Créer** (vous pouvez aussi utiliser le **+** à côté d'**Agents** dans
la barre latérale gauche).

### 2. Choisir le type Extraction

Dans la boîte de dialogue **Nouvel agent**, sous **Type d'agent**, sélectionnez
**Extraction**. Le type détermine ce que fait l'agent : choisissez-le avant de
continuer.

### 3. Nommer l'agent et le créer

Saisissez un **Nom** (au moins 3 caractères) dans le champ **Nom**, puis cliquez sur
**Créer**. Votre agent est créé avec des valeurs par défaut pertinentes et s'ouvre
dans l'éditeur.

### 4. Régler l'essentiel dans Général

L'éditeur s'ouvre sur l'onglet **Général**. Définissez :

- le **Nom** — la façon dont l'agent apparaît dans votre espace de travail ;
- la **Langue** — la langue dans laquelle l'agent travaille ;
- les **Instructions** — ce que l'agent doit extraire de chaque document.

Cliquez sur **Enregistrer**.

### 5. Choisir le modèle dans Modèle

Ouvrez l'onglet **Modèle** et choisissez le **Modèle** qui pilote l'agent, ainsi
qu'une **Température** (une valeur basse donne des extractions plus régulières et
littérales). **Enregistrez**.

### 6. Définir la sortie dans Sortie

Ouvrez l'onglet **Sortie** et construisez les champs à extraire **visuellement** —
sans JSON. Cliquez sur **Ajouter un champ** et, pour chaque champ, définissez :

- **Nom du champ** — la clé sous laquelle il est stocké (par ex. `title`).
- **Type** — **Texte**, **Nombre** ou **Oui / Non**.
- **Description** — l'indice que l'agent utilise pour décider quoi y mettre.
- **Obligatoire** — activez-le pour les champs que l'agent doit remplir.

Supprimez un champ avec son icône **corbeille**. Vous préférez éditer le schéma à la
main ? Cliquez sur **Mode avancé** (en haut à droite) pour passer à un éditeur JSON
brut, et **Éditeur visuel** pour revenir. Cliquez sur **Enregistrer**.

### 7. Enregistrer chaque onglet

Comme chaque onglet est un formulaire à part, cliquez sur **Enregistrer** sur chaque
onglet modifié (ou **Annuler** pour abandonner les modifications d'un onglet). L'agent
d'extraction est alors prêt à s'exécuter sur des documents — voir
[Lancer une extraction](/fr/run-an-extraction).

## Historique des versions

Chaque enregistrement crée une nouvelle **version** de l'agent. Ouvrez **Historique**
(en haut à droite de l'éditeur, avec le **v numéro** actuel) pour parcourir les
versions passées dans un panneau latéral, **comparer** les changements
(**Changements de cette version** ou **Comparer avec l'actuelle**), et **Restaurer
cette version** — ce qui copie ces paramètres comme nouvelle version actuelle sans
rien supprimer. Voir [Historique des versions](/fr/agent-version-history) pour le
guide complet.

## Conseils

- Gardez des **descriptions** de champs claires dans le schéma — l'agent s'en sert
  pour décider quoi mettre dans chaque champ.
- Utilisez une **Température basse** (par exemple `0.2`) pour l'extraction : vous
  voulez une sortie fidèle et reproductible plutôt que des réponses créatives.
- Ne marquez comme **Obligatoire** que les champs réellement requis ; l'agent
  renvoie `null` pour une valeur obligatoire qu'il ne trouve pas.
- L'éditeur visuel propose des champs **Texte**, **Nombre** et **Oui / Non**. Pour
  des formes plus riches (listes, objets imbriqués), passez en **Mode avancé** et
  éditez le JSON.

## Dépannage

- **Le bouton Créer est désactivé** — le **Nom** doit comporter au moins 3
  caractères.
- **Je ne vois pas le type de champ dont j'ai besoin** — l'éditeur visuel couvre
  **Texte**, **Nombre** et **Oui / Non** ; utilisez le **Mode avancé** (JSON brut)
  pour des listes ou des objets imbriqués.
- **Mon schéma ne s'enregistre pas en Mode avancé** — il doit être un objet JSON
  valide ; l'éditeur affiche *Le schéma JSON de sortie doit être un objet JSON valide*
  et vous maintient en Mode avancé jusqu'à correction.
- **Je ne vois pas d'onglet Sortie** — seuls les agents non conversationnels en ont
  un. Assurez-vous d'avoir créé un agent **Extraction** (les agents conversationnels
  discutent à la place).
- **Les valeurs extraites semblent incorrectes** — affinez les **descriptions** des
  champs et les **Instructions**, puis cliquez sur **Enregistrer** et réessayez.
